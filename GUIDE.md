## 第三部分：使用教程

### 连接钱包

1. 打开 `http://localhost:3000`
2. 点右上角 **"连接钱包"** 按钮
3. MetaMask 弹出后点 **"连接"**
4. 右上角显示你的地址（如 `0xf39f...2266`）说明连接成功

---

### 功能1：存款（Supply）

存款可以赚取利息，存入的代币也作为借款的抵押品。

1. 点导航栏 **"Supply"**
2. 点选要存入的代币（**C5D** 或 **WETH**）
3. 在金额栏输入数量（如 `1000`）
4. 点 **"Deposit"** 按钮
5. MetaMask 弹出**两次**确认：
   - 第一次：**授权**（允许合约使用你的代币）→ 点确认
   - 第二次：**存款交易** → 点确认
6. 看到 `✅ Successfully deposited` 说明成功

---

### 功能2：借款（Borrow）

借款前必须先存款作为抵押，借款金额不能超过 LTV 限制。

| 代币 | 最多可借（LTV） | 清算阈值 |
|------|----------------|---------|
| C5D | 抵押价值的 80% | 85% |
| WETH | 抵押价值的 75% | 80% |

例如：存入 1000 C5D（价值 $1000），最多可借 800 C5D 或 0.375 WETH（价值 $750）。

1. 点导航栏 **"Borrow"**
2. 查看 **"Max borrowable"** 了解最多能借多少
3. 输入借款金额（建议不超过最大值的 70%，保持健康因子安全）
4. 点 **"Borrow"** 按钮
5. MetaMask 弹出确认 → 点确认
6. 看到 `✅ Successfully borrowed` 说明成功

---

### 功能3：还款（Repay）

还款会偿还你的债务（包含累计的利息）。

1. 点导航栏 **"Borrow"**
2. 点上方 **"↩ Repay"** 标签
3. 选择要还款的代币
4. 输入还款金额，或点 **"MAX"** 一次还清
5. 点 **"Repay"** 按钮
6. MetaMask 弹出**两次**确认（授权 + 还款）→ 都点确认

---

### 功能4：取款（Withdraw）

取回你之前存入的代币（+ 赚到的利息）。

> ⚠️ 如果你有未还清的借款，取款后健康因子会下降，注意不要让 HF 低于 1

1. 点导航栏 **"Supply"**
2. 点上方 **"⬆ Withdraw"** 标签
3. 输入取款金额，或点 **"MAX"** 全部取出
4. 点 **"Withdraw"** 按钮
5. MetaMask 弹出确认 → 点确认

---

### 查看仪表盘（Dashboard）

点导航栏 **"Dashboard"** 可以看到：

| 数据 | 说明 |
|------|------|
| Total Collateral | 你存入代币的总价值（美元） |
| Total Debt | 你借款的总价值（美元） |
| Net Position | 总抵押 - 总债务 |
| Health Factor | 健康因子，**必须 > 1.0**，越高越安全 |
| Supply APY | 存款年化收益率（越高越好） |
| Borrow APY | 借款年化利率（越低越好） |
| Utilization | 资金利用率（已借出 / 总存入），影响利率 |

点右上角 **"↻ Refresh"** 刷新最新数据。

## 第四部分：常见问题

### MetaMask 报错"资金不足"但账户有余额

原因：MetaMask 缓存了旧数据。

解决：MetaMask → 设置 → 高级 → **重置账户（Reset Account）** → 再试一次

---

### MetaMask 只显示"查看提醒"，无法确认

原因：安全拦截没完全关掉，或者账户缓存问题。

解决：先重置账户，再试一次。还不行的话重启浏览器。

---

### 页面数据不更新

点 Dashboard 右上角的 **"↻ Refresh"** 按钮，或按 **Cmd+Shift+R** 强制刷新页面。

---

### 重启电脑后数据消失了

正常现象。Hardhat 是内存中的区块链，重启后数据清空。需要重新：
1. `npm run node`（启动区块链）
2. `npm run deploy`（重新部署合约）
3. `npm run frontend`（启动前端）

---

### 健康因子太低怎么办

健康因子接近 1.0 时，有被清算的风险。解决方法：
- 去 **Supply** 存入更多抵押品，或
- 去 **Borrow** → **Repay** 还掉一部分借款

## 第五部分：利率模型说明（答辩用）

本项目采用**线性利率模型（Linear Interest Rate Model）**：

```
利用率 U      = 总借款 / 总存款
借款年利率    = 基础利率 + 斜率 × U
存款年利率    = 借款年利率 × U × 90%
```

| 代币 | 基础利率 | 斜率 | 举例（U=50%） |
|------|---------|------|--------------|
| C5D | 2% | 10% | Borrow 7%，Supply 3.15% |
| WETH | 2% | 20% | Borrow 12%，Supply 5.4% |

**核心逻辑**：资金借出越多（利用率越高），利率越高，吸引更多人存款并抑制借款，自动维持市场平衡。

---

## 健康因子公式（答辩用）

```
HF = Σ（存款余额 × 代币价格 × 清算阈值）
   / Σ（借款余额 × 代币价格）

HF ≥ 2.0  → 安全
HF 1.0~2.0 → 需要留意
HF < 1.0  → 可被清算（仓位被强制平仓）
```

**为什么要超额抵押？**
价格波动时，如果借款价值超过抵押价值，协议会亏损。超额抵押 + 清算机制确保协议始终有足够资产偿付。
